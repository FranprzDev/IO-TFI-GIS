# Especificaciones del Simulador

Esta carpeta centraliza las specs funcionales y técnicas antes de implementar código.

## Flujo de trabajo

1. Definir y ajustar alcance en `PRD-1.md`.
2. Derivar historias/tareas técnicas a partir de cada sección.
3. Implementar por incrementos cortos (MVP -> V1 -> V2).
4. Volver a la spec y actualizar decisiones/hipótesis.

## Convenciones

- Cada cambio importante debe reflejarse en la sección de `Versionado` del PRD.
- Las decisiones de modelado estadístico deben incluir hipótesis y fuente de datos.
- No programar features sin criterio de aceptación explícito en la spec.
